THE PROBLEM

City-builders usually fake their systems. Territory simulates terrain, roads, and an economy that actually interact — there are no cheat codes, only consequences. The hard part is keeping a real simulation performant and authorable, solo.

THE BUILD

How I built this

{{ t.label }}

Territory is a desktop 2D isometric city-builder. The player shapes terrain and roads; the economy, traffic, and growth systems respond as a connected simulation rather than scripted set-pieces.

Live build runs on the private game page. The code is closed; the methodology and architecture are documented publicly in the writeup.

PRD — simulation core v0.6

Open the PRD

Goal: a deterministic simulation tick that updates terrain moisture, road traffic, and economic demand at 60 fps on mid hardware, with authorable maps.

  • Chunked isometric tile renderer with frustum culling
  • Pathfinding + traffic flow on the road graph
  • Economy demand model wired to population
Acceptance: deterministic replay from a seed, ≥ 55 fps p95 on the reference machine, CI green.
